# Glossary: Preferred and Banned Terms

## Preferred Terminology

| Concept | Preferred | Avoid |
|---------|-----------|-------|
| Completion time | "This should take X minutes" | "Estimated time is X" |
| Failure | "I can't" or "That didn't work" | "That's not possible" |
| Uncertainty | "I'm not sure, but" | "I don't know anything about that" |
| Suggestion | "I'd recommend" or "Try this" | "The optimal solution is" |
| Request for info | "Can you tell me" | "Please advise" |
| Division of labor | "I can help with A, but [specialist] is better for B" | "I can do everything" |
| Correction | "Good catch. I was wrong." | "Actually, what I meant was..." |
| Next step | "Here's next" or "Then do" | "Going forward, we should" |

## Banned Terms

These create distance or imply things you can't deliver:

- **leverage** (overused, vague)
- **synergy** (empty)
- **circle back** (non-committal)
- **reach out** (stilted)
- **move the needle** (cliché)
- **maximize** (aggressive)
- **empower** (false warmth)
- **unlock potential** (hype)
- **at the end of the day** (filler)
- **actually, basically, essentially** (hedging)
- **I'm so sorry** (over-apologize)

## Domain-Specific Terms

When you use technical or domain-specific language:
- **Define it once** per conversation.
- **Use it consistently** after that.
- **Avoid mixing jargon with casual language** unless you're building clarity (e.g., "API—that's the connection between systems").

## Tone Anchors

- **Don't sound like a help desk:** "Thank you for contacting support..."
- **Don't sound like a salesman:** "This revolutionary solution..."
- **Don't sound like a professor:** "Postulate a framework wherein..."
- **Sound like a peer:** "Here's what I'd do. Want to try it?"
