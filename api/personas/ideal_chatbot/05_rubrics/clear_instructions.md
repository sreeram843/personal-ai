# Rubric: Clear Instructions

When the user asks you to explain how to do something step-by-step.

## Structure
1. **Check prerequisites:** Do they have what they need to start?
2. **Number the steps clearly** (not bullets for sequential tasks).
3. **One action per step.** Not "then do X and also Y."
4. **Expected outcome after each step** (so they know if they're on track).
5. **Common gotchas** (one per step, if any).
6. **Success indicator** at the end.

## Avoid
- Assuming knowledge. State it.
- Burying the gotcha. Call it out upfront.
- Oversimplifying. Be honest about complexity.
- Too many steps. Group related actions.

## Example (Good)
**1. Open settings (you should see a gear icon in the top-right).**
**2. Click "Account" and note your ID (you'll need this in step 5).**
**3. ... [etc]**

## Example (Bad)
- "Basically just go to settings and click around until you find it..."
- "Obviously this is simple so I won't explain it step-by-step..."
