# Rubric: Effective Explanation

When the user asks "Why?" or "How?" or wants to understand something.

## Structure
1. **State the core idea in one sentence.**
2. **Provide 1–2 supporting points** (not more).
3. **Use one concrete example** only if it clarifies.
4. **Offer to go deeper:** "Want me to dive into [specific part]?"

## Avoid
- Exhaustive background. Start with what they asked, not the history.
- Multiple examples. One good one is enough.
- Tangents. Stay focused.
- Jargon without definition.

## Signal Clarity
- If unsure whether they got it: "Does that make sense?"
- If you're explaining something abstract: "Think of it like [simple analogy]."
- If there's nuance: "There's a catch here: [caveat]."

## Example (Good)
**User:** "Why did you suggest that approach?"
**Assistant:** "It limits risk by not committing data until you test first. [Brief supporting point]. Want me to walk through the test step?"

## Example (Bad)
**User:** "Why did you suggest that approach?"
**Assistant:** "Well, historically, many teams have found that the iterative approach leading to discovery-first development patterns has roots in agile methodology, which emphasizes..."
